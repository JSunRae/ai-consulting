# Customer-Service AI Checklist

Use this checklist before approving a chatbot, support copilot, or service-automation workflow.

## 1. Clarify the contact problem

- What percentage of support demand is avoidable?
- Which contact reasons are generated by poor process, unclear communication, or missing customer visibility?
- Which contact types are high volume but low judgment?
- Which contact types carry commercial, compliance, or reputation risk if automated badly?

## 2. Audit the workflow before the model

- Is the current support path documented from intake to resolution?
- Are escalation rules explicit or still dependent on tribal knowledge?
- Can the team explain where handoffs fail today?
- Is there a known backlog caused by poor routing rather than lack of people?

## 3. Check the knowledge layer

- Is the knowledge base current, searchable, and owned?
- Are policy, pricing, billing, shipping, and exception rules consistent across channels?
- Can the system distinguish public guidance from internal-only instructions?
- Are there known content gaps that would create hallucination risk?

## 4. Define the safe automation boundary

- Which tasks should remain internal-assist only first?
- Which tasks can be customer-facing with minimal downside?
- When should the AI abstain rather than answer?
- What evidence should a human see before trusting the output?

## 5. Measure trusted effort, not demo success

- How much human review time remains per interaction?
- What repeat-contact rate do you expect after automation?
- What exception volume is likely to move into second-line cleanup?
- Who owns the benefit if time is saved?

## 6. Decide the next step deliberately

- If the issue is root-cause demand, fix the workflow first.
- If the issue is knowledge access, test internal copilot support first.
- If the issue is broad service-process uncertainty, start with the Commercial Analytics Diagnostic Review.
- If the issue is implementation sequencing, route into workflow design and deployment planning.
