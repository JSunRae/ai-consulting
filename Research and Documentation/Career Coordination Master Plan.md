## Alignment findings

### 1. Seniority signal is too mixed

The resume PDFs are already aiming at Director / Head-level roles, but the website still leads heavily with `Senior Data Analyst EMEA`. That helps credibility, but it weakens Director-level positioning when it is the first label everywhere.

Recommended fix:

- Keep `Senior Data Analyst EMEA at Medline` as proof.
- Stop using it as the primary brand label everywhere.
- Lead with `Commercial Analytics & Applied AI Leader` on LinkedIn, GitHub, and the website hero.

### 2. Experience length is inconsistent

There is a live/public `13+ years` narrative and a local-repo `18+ years` narrative.

Current conflict observed:

- Live website home/about/resume: `13+ years`
- Local `resume.html` and `about.html` include dynamic `18+` references
- Resume PDFs: `13+ years`

Recommended fix:

- Pick one rule and use it everywhere before the next website deploy.
- Best option:
  - `13+ years across sales leadership, commercial analytics, and applied AI`
- Only use `18+ years` if you explicitly mean total post-ADFA professional history and you are willing to defend that interpretation consistently.

### 3. Live website and local repo are not fully aligned

As of 2026-05-08, the live site still shows the older `13+ years` resume framing while the local repo contains newer `18+` dynamic copy in some files. If you deploy from the current local state without deciding the canonical number first, you will create visible drift.

### 4. Website audience is split between consulting buyers and employers

The site is mostly optimized for consulting buyers, but the resume page and some copy also target hiring managers. That is workable, but right now the split is implicit rather than controlled.

Recommended fix:

- Keep consulting as the primary site objective.
- Add a clear secondary path for employers and recruiters:
  - `Hiring me for a leadership role? Start here.`
- Make the split explicit on the homepage and resume page.

### 5. GitHub profile is materially out of alignment

The public GitHub profile currently says:

- `New to learning python and ML.`

That is directly contradictory to the website, resume, and portfolio narrative, and it will damage credibility with recruiters or hiring managers who click through.

### 6. LinkedIn appears under-positioned

The public snippet shows Medline, Stuttgart-area location, and a German-language intro beginning with `Als visionärer Senior-Datenanalyst...`. That likely undersells the Director / Head track and is not tightly aligned to the current website and resume positioning.
