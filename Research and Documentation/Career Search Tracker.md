## Canonical packet system

- Every role starts from a lane-specific base variant, not from a one-off prior packet.
- Canonical workflow:
  1. Verify the role is still live and that location / work-authorization fit is real.
  2. Score it on scope, industry, analytics-vs-platform tilt, compensation confidence, work-auth risk, and stretch type.
  3. Pick the closest base resume variant first.
  4. Tailor only the summary, top proof points, role-alignment table, and cover letter.
  5. Create a packet containing fit memo, role-specific resume, role-specific cover letter, form-answer deltas, and blocker note if needed.
  6. Log the action in `Application Submission Log.md` on the same day.
